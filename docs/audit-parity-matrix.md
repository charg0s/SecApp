# Audit Parity Matrix

Status: research-backed design matrix, not runtime proof
Research date: 2026-07-18
Pinned release tree: [v0.77.1](https://github.com/Velocidex/velociraptor/tree/v0.77.1)

Resolved commit: [3137c7f714ab344dd37d0df1d5393573e41b30a5](https://github.com/Velocidex/velociraptor/tree/3137c7f714ab344dd37d0df1d5393573e41b30a5)

## Method and confidence

"Confirmed artifact" means its YAML exists in the v0.77.1 release tree and its declared name, source, parameters, dependencies, and permissions were inspected. "No dedicated artifact confirmed" means a release-tree search did not identify a purpose-specific built-in; it is not a claim about later or community artifacts.

VQL candidates are grounded in the official [VQL reference](https://docs.velociraptor.app/docs/vql/vql_reference/), including [Windows plugins](https://docs.velociraptor.app/vql_reference/windows/), [`wmi()`](https://docs.velociraptor.app/vql_reference/windows/wmi/), [`authenticode()`](https://docs.velociraptor.app/vql_reference/windows/authenticode/), [parsers such as `parse_evtx()`](https://docs.velociraptor.app/vql_reference/parsers/), and [common plugins such as `glob()`, `netstat()`, and `pslist()`](https://docs.velociraptor.app/vql_reference/popular/). Candidate Windows namespaces, keys, and access levels still require tests on the supported Windows matrix.

"Offline: yes" means the reviewed query has no intentional external-service requirement. It is not a network sandbox and does not prevent unrelated host traffic.

Required/implied permissions are metadata, not an enforcement boundary. The matrix treats imports, Artifact calls, VQL functions, optional branches, and pinned parameters as the transitive capability surface.

## Custom artifact policy

Custom artifacts listed above are design names only. Before use, each must:

1. have a unique non-upstream name and pinned content digest;
2. declare Windows preconditions and the complete transitive capability set;
3. use no `execve()`, network, write, upload, remediation, or memory capability in the default profile;
4. bound rows, bytes, runtime, and target scope;
5. classify sensitive columns before normalization;
6. pass the official `artifacts verify` static check ([artifact CLI](https://docs.velociraptor.app/docs/cli/commands/artifacts/));
7. pass synthetic positive, negative, unsupported, permission-denied, timeout, and malformed-input tests;
8. be re-reviewed against every supported release tree and the separately verified runtime binary pin;
9. fail closed when an import, Artifact call, VQL function, optional branch, or parameter cannot be classified.
