# Security Policy

SecApp is at an early design stage and has no supported release yet.

Do not place vulnerability details, credentials, evidence, personal data, or system data in a public issue. If GitHub private vulnerability reporting is available for this repository, use it. Otherwise, open a minimal public issue asking the maintainers to establish a private reporting channel, without including sensitive details.

Reports should identify the affected revision, describe the impact, and provide the smallest safe reproduction possible. Never attach real forensic evidence or data belonging to another person or system.
